Writing guidelines

We use these guidelines to record our best practices and patterns for all text-based messaging in our products.

Writing style

We write in active standard voice. Write as if you are speaking directly to another person. Use verbs to start a statement and describe the action the user performs.

For example:

  • Select a property, not A property should be selected.
  • Click this
  • Select x from the list
  • Choose property

Six great guidelines for clear, concise writing:

  1. Don't use a metaphor, simile, or other figures of speech that you commonly see.
  2. Don't use a long word if a shorter one can convey the same message.
  3. If you can omit a word, do.
  4. Use active voice.
  5. Don't use technical terms, scientific nomenclature, or jargon if there's an everyday word you can use instead.
  6. Be direct and to-the-point.

Before you begin: Identify your audience

Identifying your audience ensures that you write clearly and helps you focus on the End-User's needs.

Examples of audiences:

  • Users who set up and use ARGUS APIs
  • New users
  • User admins
  • Admins with assumed skills/permissions

British English vs American English:

To maintain consistency with our public marketing sites and other comms, we favor American English. This means we favor using z over s where we have an option.

For example: organize, instead of organise.

Capitalization

Sentence-case capitalization makes it easy for readers to distinguish between common nouns and proper nouns and is generally considered the quickest to read. Use sentence-case capitalization for all UI elements where it is an option.

For example:

  • Tenant reports
  • Rent Roll window
  • Cash Flow Report

We do not consider UI elements within a product to be proper nouns. Nor do we support a concept of important or special words. Determining whether something should be 'important' or not is subjective and could lead to inconsistency across the platform.

We always use sentence-case capitalization, unless the name is for a product or service or a user role such as 'Administrator'.

Active voice

  • Do: Administrators control user access to ARGUS Cloud applications.
  • Don't: User access to ARGUS Cloud applications is controlled by administrators.

Bold text

Use bold text to draw the users attention to a key phrase or statement in your content. If you want to bold an element but the UI doesn't allow it, please speak to a designer.

  • Do: Go to User Settings > Users.
  • Don't: Go to the settings area and click users.

Italics

Use italics for emphasis, citations, or defining new terms. Do not use italicstext that is ahyperlink.

Interjections

Don't use interjections in customer facing copy.

  • Don't: oops, phew, shucks, woops, yikes, dang

Punctuation

General guidelines:

  • Introduce bulleted lists with a colon.
  • Don't use commas at the end of list items in bulleted or numbered lists.
  • If any list item contains two or more sentences, all list items should be punctuated.
  • If all list items are one sentence, don't punctuate.
  • Don't put a comma or semicolon at the end of a bulleted or numbered list item.

Colons

Use colons to precede a bulleted list or series of numbered steps. Dont use colons on headings.

Semicolons

Avoid using semicolons if possible. If you really need them, use semicolons to:

  • Connect two closely related ideas, if they are both independent clauses (full sentences that could stand on their own).
  • Replace a comma or to replace the word and between two closely related ideas.

Do: My friends made me cookies on my birthday; I've never been happier.

Don't: Cannot, can not, it is

Excamation marks

Avoid the use of exclamation marks, they should only be used for exciting new things!

Full stops (periods)

Use only one space after a full stop. Avoid using them in headers, titles, tooltips, labels and menus. Use them in descriptions, messages, alerts and notifications. Don't use them in a list unless the point is a complete sentence.

Lists

Use lists to break your content into scannable chunks. Use proper punctuation if your list items are complete sentences. Try to limit lists to a maximum of ~6 items. If you need more items, try splitting it into multiple lists.

Bullet lists

Use bulleted lists when the order of the list items is not a priority. If the bullets complete an introductory sentence, start each item with lower-case and don't include periods.

Do

Due to security issues, users are now required to:

  • Do: Go to User Settings > Users.
  • Don't: Go to the settings area and click users.

Don't

Due to security issues, users are now required to follow the steps outlined below:

  • Do: Go to User Settings > Users.
  • Don't: Go to the settings area and click users.

Numbered lists

Use numbered lists when the order of the list items matters. Unlike bulleted lists, always capitalize the first word in each item and end with a period.

Do

To add a new user:

  • Go to Settings > User settings > Users.
  • Click the Add button.
  • Enter the user's details.

Titles and headings

Use sentence-case for headings. Don't use italics, or standard punctuation.

  • Capitalize the first word (sentence-case)
  • Capitalize proper nouns and trademarked names (products, countries, people's names etc...)

Gender pronouns

Use sentence-case for headings. Don't use italics, or standard punctuation.

  • Do: Ask your admin for assistance.
  • Don't: Ask your admin if he or she can assist you.

Numbers

Spell out numbers up to ten, one, two, three. After ten we can use numerals, 11, 121, 768 and so forth.

Oxford comma

We can use the Oxford or serial comma for the final item in a list.

  • Do: We use sentence case in all titles, headings, menu items, and buttons.
  • Don't: We use sentence case in all titles, headings, menu items and buttons.

Ampersands

Don't use ampersands (&). They attract attention to the least important part of the sentence. Spell out the word and.

Abbreviations

In general, don't use internal abbreviations in customer-facing copy. Don't use apostrophes in plural abbreviations.

Don't use i.e. or e.g.; they do not play nice with localization.

  • Do: We use sentence case in all titles, headings, menu items, and buttons.
  • Don't: We use sentence case in all titles, headings, menu items and buttons.

Product name abbreviations

In general, don't use internal abbreviations in customer-facing copy. Don't use apostrophes in plural abbreviations.

Approved product names

Use the full product name where space allows.

For example:

  • Headings
  • Titles
  • Descriptions
  • Messages
  • Alerts

Use abbreviations only where space is tight.

For example:

  • Column headers in tables (not neccessary if resizing columns is enabled)
  • Labels on components

Onboarding new users

When onboarding new users via Fusion spotlight modals, we need to be mindful of how we communicate information to our users.

For titles:

Try to state the potential benefit of a feature for the user, as opposed to simply naming or describing the feature.

For descriptions:

Try to be specific about the features we support and present them in a succinct, scannable way. Do not assume that the user will read a long paragraph of text in full.
